## Recall Bench Docs — Cast Creation + 8+1 Dimensions

Updated `docs/recall-bench.md` with three substantial additions per stevenic's request.

### Reframe scoring as "The 8+1 Dimensions"

Replaced the old "Phase 3 Scoring" mini-table + standalone "Evaluation Dimensions" section with a unified "The 8+1 Dimensions" section. Structure:

- **Intro + ASCII diagram** showing 8 recall dimensions (correctness 0–3 + completeness 0–2 = recall 0–5) + 1 hallucination dimension (0–1) → composite max 6.0
- **The 8 Recall Dimensions** — preserved the existing 8-category table (factual-recall through negative-recall)
- **The +1: Hallucination** — explains why hallucination is held independent (binary, can dissociate from correctness; `negative-recall` is the canary category)
- **Tracking Hallucinations Over Time** — six concrete strategies:
  1. Hallucination row in heatmap (trajectory across corpus growth)
  2. Per-category breakdown (negative-recall + synthesis are highest-risk)
  3. Trend slope, not just absolute rate (regression intercept vs slope)
  4. Question-level reproducibility / canary questions (diff per-question scores across runs)
  5. Hallucination-vs-recall correlation plot (catches "confidently wrong" pathology where hallucination *falls* with recall instead of rising)
  6. Per-persona splits (different domains exercise different fabrication tendencies)

### "Creating a Cast" subsection in Dataset Generation Pipeline

New subsection documents the recent robustness improvements in `cli.ts`, `generator.ts`, and `cli-generator-model.ts`:

- **Incremental durable progress** — `onDay` fires per day during runArcPass/runGapPass (not at end of `generateAll`); progress indicator shows "(N unique)" count
- **Per-day failure resilience** — try/catch around each `model.complete()` call; failures log to stderr (`[generator] arc=X day=N skipped: ...`) and the run continues instead of aborting
- **Coding-agent quirks** — `claude`/`codex`/`copilot` don't accept `--temperature` or `--max-tokens`; pipeline strips them before invocation
- **Subprocess termination on Windows** — explicit watchdog + `taskkill /T /F /PID` because `cmd.exe` ignores SIGTERM; `--timeout` default for `generate` bumped to 600s (10 min)
- End-to-end example showing resume-after-skip workflow

### Stale flag fixes

- Conversation generation example previously showed `--days` and `--out` flags that no longer exist in `cli.ts` (they're hardcoded to `<persona-dir>/memories` and `<persona-dir>/conversations`). Removed those flags and added a clarifying sentence about the directory convention.
- CLI Reference snippet for `create-persona` was using `--out` (also wrong); changed to `--persona <dir>` to match the actual command.
