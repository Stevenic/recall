## Recall Bench Spec v0.1

Created `specs/recall-bench.md` — initial design spec for "Recall Bench", an agent memory benchmark system.

**Key design decisions:**
- Synthetic agent personas with 1,000 days of daily memories each
- 8 evaluation categories: factual recall, temporal reasoning, decision tracking, contradiction resolution, cross-reference, recency bias resistance, synthesis, negative recall
- 200 Q&A pairs per persona, grounded in specific relevant days
- System-agnostic adapter interface — any memory system can participate
- Judge model scores on 3 dimensions: correctness (0–3), completeness (0–2), hallucination (0–1)
- Generation pipeline: arc planner → day generator → consistency checker → Q&A generator → validation
- Narrative arcs (15–25 per persona) create temporal complexity: projects, incidents, decisions, learning, corrections

**Open questions logged:** synthetic vs. semi-real data, judge model selection, compaction evaluation, multi-turn queries, cost tracking

**Files changed:** `specs/recall-bench.md` (created), `.teammates/scribe/GOALS.md` (updated)

## Recall Bench Spec v0.2 — Cross-Domain Persona Expansion

Updated `specs/recall-bench.md` to broaden the persona catalog well beyond software engineering.

**Key changes:**

- Added design principles for persona selection (domain diversity, memory pattern diversity, arc type diversity, terminology spread)
- Updated success criteria, report example, and config to reflect 5 personas

## Recall Bench Spec v0.3 — Time-Range Subsetting & Heatmap Reporting

Updated `specs/recall-bench.md` to v0.3 per @beacon's handoff after implementing the harness.

**Key changes:**
- Added §5.4 Time-Range Subsetting: 5 named ranges (30d, 90d, 6mo, 1y, full) with fresh adapter lifecycle per range
- Updated §5.1 and §5.2 to reference range filtering — only eligible Q&A pairs (all `relevant_days` within cutoff) are evaluated
- Added §7.1 Heatmap Report: category × range matrix as primary degradation visualization, with `HeatmapGrid` TypeScript interface
- Added Q&A pair scaling guidance: recommended 300–350 pairs/persona with per-range minimums (30→30, 90→60, 6mo→100, 1y→150, full→200)
- Updated §6.3 config to show `ranges` list and `heatmap` output flag
- Updated success criteria: 300+ Q&A pairs, heatmap validation, range subsetting correctness
- Bumped version to 0.3 with changelog entry

**Decision rationale:** Time-range subsetting shows how performance degrades as corpus grows — critical for evaluating compaction-based systems. Heatmap gives a single view of which categories break down at which scale.

**Arc type distribution across all personas (107 total arcs):**
- Project: 25 (long-running multi-phase work)
- Incident: 15 (short acute events with dense facts)
- Decision: 17 (multi-phase choices with reversals)
- Learning: 13 (skill progression with early details that fade)
- Relationship: 14 (cross-arc connective tissue)
- Correction: 23 (wrong→right belief pairs with specific days)

**Design decisions:**
- Each persona.yaml includes rich metadata (profile, communication style, projects) for the generation pipeline, beyond the minimal id/name/epoch the harness reads
- Each arcs.yaml includes type, title, description, and (for corrections) wrongDay/correctedDay/wrongBelief/correctedBelief — extra fields ignored by current TypeScript interface but needed by generation pipeline
- Max 4 concurrent arcs per persona (per §4.2 constraint)
- Quiet periods (vacations, breaks) designed into each timeline
- Correction arcs have sufficient gap between wrong belief and correction (50-500 days) to let wrong info "settle"
- Cross-references between arcs are documented in descriptions to guide the day generator
