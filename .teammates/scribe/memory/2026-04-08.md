## Hierarchical Memory Spec — v0.1 Draft

Created `specs/hierarchical-memory.md` — full design spec for hierarchical (eidetic) memory architecture.

**Key design decisions:**
- Raw memories are **never deleted** — permanent source of truth (~5 MB for 3 years)
- Parent nodes (weekly/monthly) store `pointers` array in YAML frontmatter + generated summary
- Two embeddings per parent: aggregated child embedding (`#agg`) and summary embedding (`#summary`)
- Aggregated embedding = normalized mean of child embeddings (uniform weighting for MVP)
- Two-phase recall: Phase 1 (high recall — parent vector + raw vector + BM25 in parallel), Phase 1b (expand pointers recursively), Phase 2 (rerank with hybrid scoring: embed + BM25 + parent boost)
- Phase 2 scoring: `w_embed=0.5, w_bm25=0.3, w_parent=0.2` (configurable)
- Parent oversampling (default K=10) to compensate for aggregation washout
- Summaries included in results as separate type (gap coverage, not authoritative)

**New interfaces:**
- `MemoryIndex.getEmbedding()` and `upsertEmbedding()` for aggregated embedding support
- Extended `SearchResult` with `isSummary`, `parentUri`, `scoreBreakdown`
- Extended `QueryOptions` with `parentCandidates`, `rawCandidates`, `scoringWeights`, `enableBM25`
- New `HierarchicalMemoryConfig` for all tuning knobs
- Removed `dailyRetentionDays` and `weeklyRetentionWeeks` from `CompactionConfig`

**Implementation sequencing:** 5 phases (A–E), A+B parallel, D parallel with C, E anytime after A.

**5 open questions logged** — BM25 implementation choice, salience weighting signals, max expansion depth, summary result type, cross-hierarchy pointer strategy.

**Files created:** `specs/hierarchical-memory.md`

## Hierarchical Memory Spec — v0.2 (Open Questions Resolved)

Updated `specs/hierarchical-memory.md` to v0.2 per stevenic's decisions on all 5 open questions.

**Decisions applied:**
1. **BM25** — Use Vectra's built-in BM25 support, no additional library
2. **Salience weighting** — Defined 3-signal model (token count 0.4, entity density 0.3, decision markers 0.3); deferred to v2, MVP uses uniform
3. **Max expansion depth** — Always expand to leaf, no depth cap. Added **temporal affinity** scoring: multiplicative date-proximity boost when query contains time references, no recency decay so old memories are never penalized
4. **Result type** — `ResultType` enum (`RAW` / `SUMMARY`) replaces `isSummary` boolean
5. **Cross-hierarchy pointers** — Group/team nodes point directly to raw memories

**New spec content:**
- Temporal affinity formula: `exp(-|date_i - date_ref| / σ)` with configurable σ (default 30 days)
- Temporal affinity as multiplier on final score (neutral when no time reference detected)
- `temporalSigma`, `temporalAffinity` added to `HierarchicalMemoryConfig`
- `temporalReference` added to `QueryOptions`
- New acceptance criteria for temporal affinity behavior
- 2 new open questions: temporal reference extraction sophistication, salience signal extraction timing

**Files changed:** `specs/hierarchical-memory.md`

## Hierarchical Memory Spec — v0.3 (Temporal Extraction Resolved)

Updated `specs/hierarchical-memory.md` to v0.3 per stevenic's decision on temporal reference extraction.

**Decision applied:**
- **Temporal extraction** — Regex-first approach. Deterministic pattern matching for explicit dates, relative references, and quarters. LLM-assisted extraction deferred unless recall quality suffers.

**New spec content:**
- §4.7 Temporal Reference Extraction — full pattern table (explicit year, year-month, full date, relative week/month/year, quarter, named periods)
- Resolution rules: midpoint for ranges, most-specific wins for multiple references, caller override via `QueryOptions.temporalReference`
- 4 new acceptance criteria for temporal extraction behavior
- Open question #1 resolved → 1 remaining open question (salience extraction timing)

**Files changed:** `specs/hierarchical-memory.md`

## Hierarchical Memory Spec — v0.4 (Salience Promoted to MVP)

Updated `specs/hierarchical-memory.md` to v0.4 per stevenic's decision to build salience now.

**Decisions applied:**
- **Salience promoted to MVP** — no longer deferred to v2
- **Extraction timing** — Compaction-time (amortized), scores stored in parent frontmatter
- **Library approach** — Hybrid: `@huggingface/transformers` NER (`Xenova/bert-base-NER`) for PER/ORG entities + regex for project names, tools, error codes, decision markers + vocabulary `Set` for known tools. Zero new dependencies.
- **Default aggregation strategy** changed from "uniform" to "salience"

**All open questions are now resolved.** Spec is ready for implementation handoff to @beacon.

**Files changed:** `specs/hierarchical-memory.md`
