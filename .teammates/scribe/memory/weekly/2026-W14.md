## Memory Service Design Spec

Created `specs/memory-service.md` — full design spec for the recall agent memory service.

**Key decisions:**
- Four pluggable abstractions: Storage, Embeddings, Index, Model
- Re-export Vectra's `FileStorage` and `EmbeddingsModel` interfaces (don't reinvent)
- New `MemoryIndex` interface to abstract over Vectra (allows SQLite swap later)
- New `MemoryModel` interface for LLM-powered compaction
- Default model adapter (`CliAgentModel`) spawns a CLI coding agent — no API keys needed
- Default embeddings: transformers.js (`Xenova/all-MiniLM-L6-v2`) — local, no cloud
- Compaction pipeline: daily→weekly→monthly→wisdom, with typed memory extraction
- Monorepo layout: `packages/core` + future plugin packages
- CLI name: `recall` (not `teammates-recall`)

**Open questions logged in spec:** compaction prompt tuning, auto-detect CLI agent, index isolation model, watch mode scope, monorepo vs single package.

**Files changed:** `specs/memory-service.md` (created)

## Spec v0.3 — Feedback Round

Applied three feedback items from stevenic:

1. **`MemoryModel.complete()` return type enriched** — Now returns `CompletionResult` with `text`, optional `inputTokens`/`outputTokens`, and `CompletionError` (code, message, retryable, retryAfterMs). Callers can detect rate limits and track token usage.
2. **File layout simplified** — Replaced `.indexes/<name>/` (multiple named indexes) with a single `.index/` per memory root. Multiple agents/projects = multiple memory roots. Removed `MemoryIndexProvider` interface and `--index` CLI flag. `MemoryServiceConfig` now takes an optional `index: MemoryIndex` instead of `indexProvider` + `indexName`.
3. **Language bindings added** — New `bindings/` folder with thin CLI wrappers for Python, Go, Rust, and C#. Each spawns the `recall` CLI subprocess and parses `--json` output. Python/Go in v0.2, Rust/C# in v0.3. New §9 in the spec covers design principles and binding surface.

## Monorepo Playbook Created

Created `docs/TYPESCRIPT-MONOREPO-PLAYBOOK.md` — a comprehensive monorepo adaptation of the single-package TypeScript Library Playbook (`vectra/TYPESCRIPT-LIBRARY-PLAYBOOK.md`).

**Key adaptations from single-package → monorepo:**
- Root `package.json` with `"private": true` and `"workspaces"` config (npm, yarn, pnpm variants)
- Per-package `package.json` with `"repository.directory"` field and workspace protocol for cross-deps
- Root tsconfig with shared options + `tsconfig.build.json` for project references
- Per-package tsconfigs extending root, with `references` for cross-package deps
- ESLint flat config at root using `projectService: true` for auto-discovery
- Vitest as recommended test framework (native ESM, workspace-aware)
- Dependabot entries per package directory (not just root)
- CI workflow using `--workspaces` flag for build/test/lint
- Multi-package Coveralls parallel upload pattern
- Independent versioning strategy with coordinated publish workflow
- "Adding a new package" checklist (missing from single-package version)
- Bug report template with "affected package" dropdown

**Files changed:** `docs/TYPESCRIPT-MONOREPO-PLAYBOOK.md` (created)
