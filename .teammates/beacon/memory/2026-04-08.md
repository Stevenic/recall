### Refactored Day Generator to 2-Pass Pipeline

Rewrote `DayGenerator.generateAll()` from sequential day-by-day to a 2-pass arc-by-arc approach:

**Pass 1 (arc-by-arc):** Sorts arcs by startDay, selects activity days per arc (directives + spread at intervals), generates arc-focused content. Days with content from earlier arcs get merged via LLM prompt that includes existing content.

**Pass 2 (gap-fill):** Scans each calendar week for fewer than `minDaysPerWeek` active days (default: 5). Generates routine/filler content for empty weekdays to reach the target.

**New pure functions:**
- `selectArcDays(arc, epoch)` — deterministic day selection (pinned days + interval spread; short arcs every 2 weekdays, medium every 3, long every 5; weekends skipped unless incident)
- `identifyGapDays(activeDays, start, end, epoch, target)` — finds weeks below target, picks evenly-spaced empty weekdays

**New prompt builders:**
- `buildArcUserMessage(ctx, focusArc, existingContent?)` — Pass 1 prompt with PRIMARY arc focus, background arcs, merge instructions when extending existing content
- `buildGapUserMessage(ctx)` — Pass 2 prompt for routine days, active arcs mentioned in passing only

**Config addition:** `minDaysPerWeek` (default: 5, set to 0 to disable gap filling)

**Key design change:** `recentHistory` now searches all previously-generated days (not just a sequential sliding window) since arcs generate days out of calendar order. `onDay` callbacks fire once per day in ascending order after both passes complete.

**Verification:** Build clean, lint clean, 160/160 tests pass (57 generator tests, up from 35).
