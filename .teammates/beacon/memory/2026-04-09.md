## 2026-04-09

### Implemented Hierarchical Memory Architecture (Phases A–E)

Full implementation of the eidetic memory system per `specs/hierarchical-memory.md` (v0.4). All 5 phases implemented in a single pass.

**Phase A — Eidetic Storage:**
- Removed daily/weekly retention and deletion from `Compactor.compactDaily()` and `Compactor.compactWeekly()`
- Added `pointers` array and `salience` weights to YAML frontmatter in weekly and monthly nodes
- Added `parseFrontmatter()`, `parsePointers()`, and `parseSalience()` methods to `MemoryFiles`

**Phase B — Dual Embeddings + Salience:**
- Extended `MemoryIndex` interface with `getEmbedding()` and `upsertEmbedding()`
- Implemented both in `VectraIndex` using Vectra's `LocalIndex` for raw vector operations
- Created `salience.ts` — hybrid salience extraction using `@huggingface/transformers` NER (Xenova/bert-base-NER) + regex patterns + vocabulary lookup for entity density, `gpt-tokenizer` for token count, regex for decision markers
- Salience weights computed at compaction time, stored in frontmatter, used for weighted embedding aggregation
- `_storeDualEmbeddings()` method computes L2-normalized weighted average of child embeddings (#agg) and stores summary embedding (#summary)

**Phase C — Two-Phase Recall:**
- Rewrote `SearchService` with two-phase hierarchical pipeline
- Phase 1a: Parallel parent vector search + direct raw memory search
- Phase 1b: Recursive pointer expansion (monthly→weekly→daily) with dedup
- Phase 2: Hybrid reranking with configurable weights (embed/bm25/parent) × temporal affinity
- Falls back to legacy behavior when `hierarchical.enabled === false`

**Phase D — BM25:**
- BM25 weight wired into Phase 2 scoring formula
- Actual Vectra BM25 integration (`setupBm25`/`bm25Search`) needs integration testing

**Phase E — Migration:**
- Added `recall migrate --to hierarchical [--dry-run]` CLI command
- `MemoryService.migrateToHierarchical()` backfills pointers (infers from date ranges) and computes salience for existing parent nodes

**Supporting modules:**
- `temporal.ts` — regex-based temporal reference extraction (year, month, date, relative, quarter patterns) + affinity scoring formula `exp(-|date_i - date_ref| / σ)`
- `hierarchical-config.ts` — `HierarchicalMemoryConfig` type for all tuning knobs
- Extended `interfaces/index.ts` with `ResultType` enum, `ScoringWeights`, expanded `QueryOptions` and `SearchResult`

**Key design decision:** `pointers` field removed from `DocumentMetadata` (incompatible with Vectra's `MetadataTypes` index signature which doesn't support arrays).
