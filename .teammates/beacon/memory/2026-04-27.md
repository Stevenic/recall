**Hit during implementation:**
- VirtualFileStorage normalizes paths to `/`-joined and treats `\root\...` as **not absolute** (it only checks for leading `/`). This means `path.join("/root", ...)` on Windows produces backslash paths that get stored without a leading `/`, while a literal `/root/...` in a test gets stored *with* a leading `/`. Tests that bypass the engine to read directly via `storage.readFile("/root/...")` miss. Fixed by computing test paths through `path.join` consistently.

### Reframed research-scientist persona — agent-as-narrator

Stevenic reviewed the generated `day-0001.md` and `day-0008.md` and pointed out that the logs are wrong: they read like Kenji's first-person lab notebook, but they should read like the AI agent's memory log — recording humans and other agents interacting with that agent (beacon-style). User said to "adjust the generation prompts as needed" and pointed at `.teammates/beacon/memory/` as the reference.

**Diagnosis.** The bug was structural, not just stylistic: `buildSystemPrompt` told the LLM the persona IS the human ("the log should sound like ${persona.name} wrote it"), and `persona.yaml` listed Kenji as the persona. So the LLM correctly produced a Kenji-voice diary — exactly what the prompt asked for. Fix had to touch both layers.

**Changes:**

1. **`packages/recall-bench/src/generator-types.ts`** — Added `principal?: PrincipalRef` and `cast?: CastMember[]` to `PersonaDefinition`. `CastMember` carries `name | role | kind?: 'human' | 'agent'` so the prompt can render `Sarah Kim (human)` vs `@lit-search-agent (agent)`. Made `company` optional and added `institution?` (research-scientist already used `institution`, which the old code silently dropped — would have rendered `Company: undefined` in prompts).

2. **`packages/recall-bench/src/generator.ts`** — Rewrote `buildSystemPrompt`. Frames the LLM as an AI agent (a computer program) writing its OWN memory log in third person, recording who interacted with the agent today, what they asked, what the agent produced, and what was handed off. Renders the principal block + cast block when present. Embeds an explicit required output structure (frontmatter `type: daily`, single H2 date heading, H3 topic sections, no clock-time sections, end with Outstanding/Tomorrow). Explicit "DO NOT write a first-person human diary" bullet. Reinforced the same framing in the trailing instruction shared by all three user-message builders (`buildUserMessage`, `buildArcUserMessage`, `buildGapUserMessage`).

3. **`packages/recall-bench/personas/research-scientist/persona.yaml`** — Renamed persona from "Kenji Nakamura" to **"Atlas"** (the AI lab co-pilot). Kenji moved into a new `principal:` block (with role + profile). Added a `cast:` block of 11 entries — 7 humans (Sarah Kim, Marcus, Lin, Patel, Dr. Wei Chen, Lab manager, Department chair) and 4 specialist agents (`@lit-search-agent`, `@seq-design-agent`, `@stats-agent`, `@order-agent`). Rewrote `profile` and `communication_style` to describe the agent (third-person, topic-organized sections, file/decision/handoff lists, beacon-style). Kept `id: research-scientist`, `epoch`, `team_size`, `projects`, and the stress-map comment unchanged. Arcs.yaml unchanged — relationship arcs already cover the cast.

4. **`packages/recall-bench/tests/generator.test.ts`** — Added 4 tests on `buildSystemPrompt`: AI-agent framing assertions, principal+cast rendering, no-principal-no-cast fallback, `institution` falling back when `company` is absent.
