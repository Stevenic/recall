## 2026-04-06

### Vectra 0.14.0 Upgrade

Replaced custom `FileStorage` interface/implementations with vectra 0.14.0's exported abstraction.
- **Decision:** Vectra's `ListFilesFilter` is a string union (`'files'|'folders'|'all'`), not our old object shape; `FileDetails` uses `isFolder` not `isDirectory`
- **Files:** `packages/core/package.json`, `packages/core/src/interfaces/storage.ts`, `packages/core/src/defaults/local-file-storage.ts`, `packages/core/src/defaults/virtual-file-storage.ts`, `packages/core/src/files.ts`

### Recall Bench Harness Implementation

Created `@recall/bench` package — pluggable benchmark harness for evaluating agent memory systems per `specs/recall-bench.md`. Time-range subsetting (30d–1000d), topic-based Q&A categories, heatmap output, pluggable adapter + judge interfaces, CLI (`run`, `list`, `ranges`).
- **Files:** `packages/recall-bench/` (package.json, tsconfig.json, vitest.config.ts, eslint.config.js, src/types.ts, src/dataset.ts, src/harness.ts, src/report.ts, src/cli.ts, src/index.ts, tests/harness.test.ts), root `package.json`, root `tsconfig.json`

### Day Generator Pipeline

Pass 1 of two-pass generation: `DayGenerator` sequentially generates 1,000 daily memory logs via LLM from persona + arcs.
- **Decision:** `GeneratorModel` is standalone (not reusing core's `MemoryModel`) to keep bench self-contained. All state-computation functions exported as pure functions for testability.
- **Files:** `packages/recall-bench/src/generator-types.ts`, `src/generator.ts`, `tests/generator.test.ts`, `src/cli.ts` (added `generate` command), `src/index.ts`

### Persona Creator & Conversation Generator

Pass 2 pipeline: `PersonaCreator` generates persona.yaml + arcs.yaml from free-text prompt; `ConversationGenerator` turns day logs into multi-turn conversations.
- **CLI flow:** `create-persona` → `generate` → `generate-conversations`
- **Files:** `packages/recall-bench/src/persona-creator.ts`, `src/conversation-generator.ts`, `src/generator-types.ts` (new types), `src/cli.ts` (added commands), `src/index.ts`, `tests/persona-creator.test.ts`, `tests/conversation-generator.test.ts`

### CLI Generator Model — Built-in Agent Support

`CliGeneratorModel` lets `--model` accept agent names (`claude|codex|copilot`) directly instead of JS module paths. Spawns agents as subprocesses with EPIPE/EOF protection.
- **Decision:** Independent implementation (not importing from core) to keep bench self-contained
- **Files:** `packages/recall-bench/src/cli-generator-model.ts`, `src/cli.ts`, `src/index.ts`, `tests/cli-generator-model.test.ts`

### gRPC Memory Adapter

gRPC client adapter so any language can implement `MemoryBenchService` proto and be benchmarked. Dynamic proto loading (no codegen).
- **Decision:** Default port 50052 (avoids Vectra's 50051). Proto is minimal — only 5 adapter methods + healthcheck.
- **Files:** `packages/recall-bench/proto/memory_bench_service.proto`, `src/grpc-memory-adapter.ts`, `src/cli.ts`, `src/index.ts`, `package.json` (added grpc deps), `tests/grpc-memory-adapter.test.ts`
