### Recall Bench Harness Implementation

Implemented the `@recall/bench` package (`packages/recall-bench/`) ‚Äî a pluggable benchmark harness for evaluating agent memory systems, per `specs/recall-bench.md`.

**Key features:**
- **Time-range subsetting** ‚Äî benchmark can run at 30d, 90d, 6mo, 1y, or full (1000d) cutoffs. Each range ingests only days up to the cutoff and evaluates only Q&A pairs whose `relevantDays` all fall within range.
- **Topic-based Q&A organization** ‚Äî Q&A pairs are tagged by category (factual-recall, temporal-reasoning, decision-tracking, contradiction-resolution, cross-reference, recency-bias-resistance, synthesis, negative-recall). Results are aggregated per-category per-range.
- **Heatmap output** ‚Äî structured `HeatmapGrid` type (category √ó time-range matrix) ready for visualization. Also renders as a text table in the CLI report.
- **Pluggable adapter interface** ‚Äî `MemorySystemAdapter` with 5 methods (setup, ingestDay, finalizeIngestion, query, teardown)
- **Pluggable judge interface** ‚Äî `JudgeModel` for scoring answers on correctness (0-3), completeness (0-2), hallucination (0-1)
- **CLI** ‚Äî `recall-bench run`, `recall-bench list`, `recall-bench ranges` commands with `--json` and `--heatmap` flags

**Files created:**
- `packages/recall-bench/package.json`, `tsconfig.json`, `vitest.config.ts`, `eslint.config.js`
- `packages/recall-bench/src/types.ts` ‚Äî all interfaces, constants, time-range definitions
- `packages/recall-bench/src/dataset.ts` ‚Äî loads persona datasets and filters Q&A by time range
- `packages/recall-bench/src/harness.ts` ‚Äî core `BenchmarkHarness` class
- `packages/recall-bench/src/report.ts` ‚Äî text/JSON report generation with heatmap grid
- `packages/recall-bench/src/cli.ts` ‚Äî Commander-based CLI
- `packages/recall-bench/src/index.ts` ‚Äî public API barrel export
- `packages/recall-bench/tests/harness.test.ts` ‚Äî 11 tests covering filtering, harness execution, reporting, dataset loading
