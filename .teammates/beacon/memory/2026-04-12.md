## 2026-04-12

### Dreaming System — Full Implementation

Implemented the complete dreaming system per `specs/dreaming.md` (v0.1). All four phases (A–D) shipped in one pass.

**New files created (4 modules):**
- `packages/core/src/dreaming-config.ts` — All types: DreamingConfig, DreamCandidate, DreamResult, AnalysisResult, DreamStatus, etc. Default scoring weights and thresholds.
- `packages/core/src/search-logger.ts` — SearchLogger class: JSONL search log writing, reading, time-window filtering, monthly rotation, candidate/state persistence under `.dreams/`.
- `packages/core/src/signal-collector.ts` — Five signal collectors (hit frequency, gap, entity, staleness, wisdom drift) + candidate scoring with configurable weights. Uses lightweight regex entity extraction (skips NER for speed).
- `packages/core/src/dream-engine.ts` — DreamEngine: three-phase orchestrator (gather → analyze → write). Five analysis prompt templates (cross-reference, gap, contradiction, typed-memory, theme). JSON + freeform output parsing. Writes insights, promotions, contradictions, and diary entries.

**Files modified:**
- `packages/core/src/search.ts` — Added `setSearchLogger()` method and `_logSearchResults()` hook. Every search logs to JSONL when logger is set.
- `packages/core/src/files.ts` — Added `listInsights()`, `listContradictions()`, `readDreamFile()`. Extended `initialize()` to create dream output directories.
- `packages/core/src/service.ts` — Added `DreamingConfig` to `MemoryServiceConfig`. Wired SearchLogger into SearchService when dreaming is enabled. Added `dream()` and `dreamStatus()` methods. Extended `_indexAllFiles()` to index insights and contradictions.
- `packages/core/src/cli.ts` — Added `recall dream` (full session, --dry-run, --phase, --max-candidates, --agent), `recall dream status`, and `--dream` flag on `recall watch`. CLI enables dreaming by default.
- `packages/core/src/index.ts` — Exported all new types, classes, and functions.

**Test file:**
- `packages/core/tests/dreaming.test.ts` — 29 tests covering SearchLogger (6), Signal Collector (12), and DreamEngine (11).

**Key decisions:**
- Signal collectors use lightweight regex entity extraction instead of NER model — keeps dreaming fast (NER is heavy for scanning 30+ daily logs).
- Search logging is best-effort (try/catch in _logSearchResults) — never fails a search due to logging issues.
- Analysis output parsing tries JSON first, falls back to freeform (produces low-confidence insight from raw text).
- Promoted typed memories are deduplicated against existing typed memories by filename.
- Used `path.join()` in test assertions for Windows backslash compatibility (WISDOM.md lesson applied).

**Verification:** Build clean, lint clean (0 warnings), 111/111 tests pass.
