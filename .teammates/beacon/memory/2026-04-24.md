## 2026-04-24

### research-scientist persona generation — bug fixes + smoke run

Pass 1 day-log generation for research-scientist persona produced 0 days with exit 0 (silent failure). Diagnosed and fixed multiple latent bugs in `@recall/bench`. 1-day smoke produced high-quality `day-0001.md` (14.7 KB); 30-day smoke ran in background.

**Bugs fixed:**
- **Commander coercer misuse** — bare `parseInt`/`parseFloat` as coercers got `(value, previousValue)` → `parseInt("1", 1)` returned `NaN`, survived `??` defaults, killed arc filter. Added `parseIntArg`/`parseFloatArg` wrappers.
- **`claude --print` flag mismatch** — unconditionally appending `--temperature`/`--max-tokens` made `claude` exit 1. Gated on `!isWellKnown` so built-in CLIs (`claude`/`codex`/`copilot`) skip them.
- **All-or-nothing writes** — `onDay` only fired after both passes; one timeout discarded 24 min of in-memory work. Fire `onDay` after each successful day in `runArcPass`/`runGapPass`.
- **Single failure aborted run** — added try/catch per `model.complete` call, log skip, continue.
- **Default timeout too short** — bumped `--timeout` 120s → 600s.
- **Spawn timeout doesn't kill Windows subprocess** — Node's `spawn({ timeout })` relies on SIGTERM which Windows doesn't implement. Added explicit `setTimeout` watchdog with `taskkill /PID <pid> /T /F` on Windows, SIGKILL on POSIX, plus `settled` guard against double-settling.

**Note:** token tracking shows 0/0 because `claude --print` doesn't emit token counts. Cosmetic only.

**Performance reality:** ~3–7 min/day wall-clock; full 1000-day run = 3–5+ days sequentially. Options for user: overnight batch, add Claude API mode for parallelization, or reduce per-day prompt work.

**Files changed:** `packages/recall-bench/src/cli.ts`, `packages/recall-bench/src/cli-generator-model.ts`, `packages/recall-bench/src/generator.ts`, `packages/recall-bench/tests/generator.test.ts`.
