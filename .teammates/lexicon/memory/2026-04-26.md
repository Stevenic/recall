### Task: Wiki vs typed memories question + visibility wisdom

**User question:** "do we need typed memories if we have the wiki?" (broadcast `@everyone`)

**My answer:** Keep both. They optimize different retrieval distances.
- Typed = atomic single-claim files. Sharp embedding match, low surrounding noise. Replace-on-update lifecycle.
- Wiki = topical hubs with multi-claim sections + `[[links]]`. Synthesis context, expansion entry points. Heavy lifecycle (merge/rename/lint).
- Collapsing typed → wiki causes (1) distance dilution (claim buried in topical page), (2) lifecycle mismatch (small atomic facts don't deserve hub treatment).
- Resolution: wiki pages CITE typed memories (`[ref:typed/...]`), don't absorb them. Spec this in `specs/wiki.md` §12 surface-area rule.
- Edge case: if every wiki page stays ≤ 200 tokens, the surfaces functionally merge — but then you've lost the synthesis benefit.
